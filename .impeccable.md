# impeccable — project design context

## Design Context

### Users

BlewChat is a Tauri cross-platform chat demo (desktop + iOS + Android) for the
`iroh-ble-transport` crate. The primary audience is **developers and testers**
exercising the BLE transport on real hardware, but the app is **also shown to
collaborators, stakeholders, and conference-style demos**. Users are typically
walking between devices, watching peer state, bandwidth counters, and retransmit
stats while validating that the radio layer holds up.

The job-to-be-done is: *"Prove the transport is working (or failing) at a
glance, on whichever device I'm holding, without ambient light or motion
getting in the way."* Chat is the fixture; telemetry visibility is the product.

### Brand Personality

Three words: **technical, retro, confident.**

The current direction (Press Start 2P pixel display font, hexagon `⬡` glyph,
mono IDs, deep navy surfaces with indigo primary) is intentional and should be
**leaned into**, not softened. The vibe is a radio operator's console — a
hacker-adjacent tool that takes itself seriously, exposes the wire protocol,
and doesn't pretend to be something it isn't (the first-run modal openly calls
itself "a toy" and "insecure").

Voice in microcopy is dry and a little warm ("Warming up the radio…"), never
corporate, never marketing. Emotional goal: the user feels *oriented and in
control* — they can see every peer's phase, every byte on the wire, every
retransmit count, without the UI hiding or prettifying.

### Aesthetic Direction

- **Dark mode only.** No light theme planned.
- **Palette:** deep navy (`#0b0d16`) background, layered surfaces
  (`#141728` / `#1d2138`), indigo primary (`#6366f1`), accent greens/ambers/reds
  for peer phases. Sender-name palette is a fixed 8-colour cycle.
- **Type system:** `Press Start 2P` for app/welcome titles only (pixel, display);
  `Inter` for body; `SF Mono / JetBrains Mono / Fira Code` for endpoint IDs,
  bandwidth numbers, and debug events. Tabular-nums for numeric telemetry.
- **Iconography:** hexagon `⬡` as logo mark; geometric Unicode glyphs
  (`⊞ ＋ ⎘ ✎ ✕ ↑ ↻ ✂`) over illustrative icons.
- **Motion:** a `pop` spring on message bubble entry; pulse on in-flight peer
  phases (handshaking / connecting / reconnecting). Honour
  `prefers-reduced-motion`.
- **References the feel *should* evoke:** terminal tooling, wire-protocol
  inspectors (Wireshark's density at chat scale), retro arcade HUDs, the
  `iroh` ecosystem's own graphic voice.
- **Anti-references — do NOT look like:**
  - A consumer messenger (iMessage / WhatsApp / Signal-lite skin).
  - Corporate/enterprise SaaS (bland flat-blue, over-rounded, empty whitespace).
  - A game or novelty app (mascots, heavy gradients, playful illustrations,
    sticker packs).

### Design Principles

1. **Telemetry is first-class, not tucked away.** Bandwidth, retransmits,
   truncations, peer phases, L2CAP-vs-GATT path — these are the point of the
   app. Surface them where they're visible at a glance; don't hide them behind
   settings panes.
2. **Readable under hostile conditions.** BLE testing happens walking around,
   often in sunlight, often one-handed. High contrast between foreground text
   and surfaces. Safe-area insets, large-enough touch targets, no thin grey-on-
   grey. Mobile-first, desktop second.
3. **Honest about what it is.** This is a transport demo, not a messenger.
   Keep the "(demo)" / "insecure" / "toy" framing visible where appropriate.
   Don't polish the product into something it isn't.
4. **Retro-technical, not retro-kitsch.** The pixel font and hex logo earn
   their place by being *restrained* — used for titles, not body text, not
   buttons, not everywhere. The rest of the UI is clean, dense, and
   contemporary around that anchor.
5. **Motion respects the user.** Spring bubbles and phase pulses are delight
   moments; both should gracefully disable under `prefers-reduced-motion`.
   No animation for animation's sake.
